# DroneBridge — Руководство пилота

## Схема связи

```
Дрон (ESP32) → 📱 Хотспот телефона → 4G → ☁️ VPS (DO) → 🌐 Интернет → 💻 Ноутбук

Телефон — только точка доступа. Ни Tailscale, ни VPN не нужны.
```

## Что нужно

- ESP32-S3 (встроен в дрон) с прошивкой DropCtrlV3
- Телефон с WiFi хотспотом и мобильным интернетом (4G/5G)
- Ноутбук с Mission Planner и интернетом
- VPS (уже настроен: 134.209.206.127, Amsterdam)

## Перед полётом

### 1. Включить хотспот на телефоне

- SSID: `LEO`
- Пароль: `88888888`
- Мобильные данные: **ВКЛ** (без них ESP32 не выйдет в интернет)

### 2. Подать питание на ESP32

ESP32 сам подключится к LEO и начнёт отправлять телеметрию на VPS.

Проверить: светодиод ESP32 горит синим.

### 3. На ноутбуке

Вариант А — **один клик**:

```
connect.bat
```

Вариант Б — **вручную**:

1. Открыть `scripts\vps_proxy.ps1` (держит мост между VPS и MP)
2. Открыть Mission Planner
3. Кнопка **Connect** (правый верхний угол)
4. Выбрать **UDP**
5. **Host:** `127.0.0.1` **Port:** `14552`
6. Нажать **OK**

Через 3-5 секунд зелёная полоса — связь есть!

### 4. Переключение дронов

Если летать по очереди (один за другим):

1. Выключить дрон 1
2. Включить дрон 2
3. Через 5 секунд телеметрия дрона 2 пойдёт в MP автоматически
4. SYS_ID в MP покажет какой дрон подключён

Если нужно изменить номер дрона (SYS_ID):

```
Подключи ESP32 к USB
Открой монитор порта (115200 бод)
Введи: SYSID=2
Введи: SAVE
```

## Если не работает

| Симптом | Причина | Решение |
|---------|---------|---------|
| Нет зелёной полосы | MP не нажал Connect | Нажми Connect вручную |
| Нет зелёной полосы | Телефон без 4G | Включи мобильные данные |
| Пинг 68ms + | VPS в Амстердаме | — |
| "VPS offline" | Нет интернета на ноутбуке | Проверь связь |
| Связь прерывается | 4G пропадает | Перезагрузи телефон |

## VPS параметры

- Сервер: `134.209.206.127` (DigitalOcean Amsterdam)
- Порт дрона: **14550** (телеметрия от ESP32)
- Порт GCS: **14551** (команды от MP через прокси 14552)
- Relay: systemd-сервис `udp-relay` (запущен всегда)
